## 4. 错误处理流程

### 4.1 API调用失败
```
调用Leonardo API
    ↓
失败？
    ↓ 是
重试（最多3次，间隔5秒）
    ↓
仍然失败？
    ↓ 是
使用备用方案：
  - 使用预设模板图片
  - 或简化为图片轮播
    ↓
向用户说明情况
```

### 4.2 子Agent超时
```
派发任务给子Agent
    ↓
超时？
    ↓ 是
检查进度
    ↓
卡住？
    ↓ 是
重新派发（最多2次）
    ↓
仍然失败？
    ↓ 是
主控接管，使用简化方案
    ↓
向用户说明
```

### 4.3 用户不满意
```
用户反馈不满意
    ↓
了解具体问题
    ↓
分析原因
    ↓
├─ 问题1: 文字不合适
│   └─ 调整文字内容和样式 → 重新生成
├─ 问题2: 节奏太慢/快
│   └─ 调整时长参数 → 重新生成
├─ 问题3: 风格不符合
│   └─ 更换模板 → 重新生成
└─ 问题4: 质量问题
    └─ 检查原图质量 → 提供建议 → 重新生成
```

## 5. 并行处理策略

### 5.1 可以并行的任务
```
阶段3: 生成9张分镜图
├─ Scene 1-3: 并行生成（Leonardo API）
├─ Scene 4-6: 并行生成（Leonardo API）
└─ Scene 7-9: 并行生成（Leonardo API）

时间: 串行需要15分钟 → 并行只需5分钟
```

### 5.2 必须串行的任务
```
创意分析 → 分镜设计 → 图片生成 → 提示词生成 → 视频合成
（每个阶段依赖前一个阶段的输出）
```
